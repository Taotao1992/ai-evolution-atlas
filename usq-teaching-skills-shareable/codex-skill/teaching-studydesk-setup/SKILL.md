# Teaching StudyDesk Setup

## Overview

Use this skill for the StudyDesk shell-setup workflow shown in the UniSQ recording, especially when an integrated shell has to be adapted to a shorter teaching pattern such as a 7-week offering with a flexible period. The guide focuses on checking release state first, aligning schedule settings with institutional data, then moving or combining weekly content carefully.

## Workflow

1. Confirm the shell and offering pattern.
   Identify the exact course shell, the teaching period, whether the shell was integrated from an earlier semester, and the target structure such as 7 teaching weeks plus one flexible period.

2. Review the reference guide.
   Read [references/panopto-guide.md](references/panopto-guide.md).
   The reference includes workflow notes extracted from the actual recording.

3. Check what students can already see.
   The recorded guide treats this as urgent after integration because Orientation Week content may already be visible.
   Review `Getting Started`, Orientation materials, visible sections, and obvious student-facing links before editing anything else.

4. Lock down not-ready material first.
   Hide assessments, practice exams, and other not-ready activities before doing the structural tidy-up.
   Use student view or an equivalent learner preview to confirm what is still visible.

5. Align schedule settings with the institutional pattern.
   Check the StudyDesk schedule and the course administrator settings.
   The recording explicitly warns that both places must agree on the number of weeks and flexible learning periods.
   If PeopleSoft or the institutional schedule says a different pattern, follow the institutional source instead of guessing.

6. Rebuild the week structure carefully.
   Move or combine content from the longer-shell structure into the shorter delivery pattern one week at a time.
   Put course-introduction material and the introductory recording in Orientation Week.
   Combine or move later-week topics so the visible structure matches the new week count.
   If a weekly section name becomes inaccurate after moving content, rename it clearly.

7. Check dates and release sequencing.
   Verify quiz due dates, assessment dates, timetable or consultation items, and any week labels that were copied from the older shell.
   Release Orientation materials early, but do not release every later week immediately.
   The recording suggests releasing only the initial weeks first, then later weeks after the structure is checked.

8. Communicate and verify.
   Send or prepare the Orientation announcement telling students what to watch and do first.
   Re-check the shell in student view and confirm the visible weeks, schedule, and due dates match the intended offering.

## Safety Checks

- Stop before bulk-releasing all weekly content in an active shell unless the user explicitly wants that.
- Do not guess the number of weeks, flexible periods, or date pattern; match the institutional source.
- Treat `course administrator` settings and `schedule` settings as a paired change and verify both.

## Execution Notes

- Use the Computer Use plugin for live StudyDesk actions.
- If the request is specifically about project groups or group-member import, prefer the separate `studydesk-group-import` skill.
- If the shell is already live, mention which changes immediately affect students before applying them.
