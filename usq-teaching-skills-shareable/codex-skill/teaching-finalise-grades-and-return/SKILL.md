# Teaching Finalise Grades And Return

## Overview

Treat grade finalisation and examiner return as high stakes and confidential. The recorded workflow is: audit the StudyDesk gradebook first, export to Excel for distribution and anomaly checks, preview the export before PeopleSoft transfer, manually resolve special result-code cases, then approve examiner return and distribution sections class by class.

## Workflow

1. Confirm authority, scope, and confidentiality.
   Identify the course, class/offering, deadline, and whether the task is audit-only or includes live approval.
   Treat any visible student results as confidential and avoid exposing them in shared output.

2. Review the reference guide.
   Read [references/panopto-guide.md](references/panopto-guide.md).
   The reference includes workflow notes extracted from the actual recording.

3. Reconcile the StudyDesk gradebook first.
   Confirm exam marks are correct, including the latest attempt where students had multiple exam attempts.
   Check that obvious overrides, late-penalty effects, and missing marks are understood before exporting anything.

4. Export and audit the grades in a spreadsheet.
   Export the grade report to Excel before the PeopleSoft step.
   Sort and scan for missing marks, unreasonable patterns, and grade-distribution issues.
   The recording explicitly checks high-grade distribution and borderline students near key pass or grade thresholds.

5. Investigate anomalies before transfer.
   Spot-check the preview/export against the StudyDesk gradebook for at least one student.
   Pay extra attention to students who are close to pass, supplementary, or a missing-assessment decision.
   If a student has strong performance except for one missing item, verify whether that item is truly missing before finalising the fail outcome.

6. Handle manual result-code cases carefully.
   After transfer, the recording distinguishes manual result-code cases such as `IAS` and `IM` that are not fully automatic.
   Apply these only after checking the local rule about exam pass/fail and assignment completion.
   Save after each manual correction batch.

7. Complete examiner return class by class.
   Open the examiner-return area, double-check the populated values, then approve only when the class is fully correct.
   Repeat for each class/offering rather than assuming one approval covers all classes.

8. Complete distribution/significance sections.
   Fill in the required distribution questions and commentary.
   The recording specifically checks whether the HD/A share is acceptable and whether distribution has improved relative to prior offerings.
   Save each section before leaving it.

9. Pause before final approval.
   Immediately before approval, reconfirm that all classes, manual codes, and distribution notes are correct.

## Safety Checks

- Never assume hurdle, rounding, special-code, or supplementary policy if the current institutional rule differs from the recording.
- Do not approve examiner return or the final distribution step while unresolved anomalies remain.
- If the user asked only for an audit, stop at the ready-to-approve state.

## Execution Notes

- Use spreadsheet tooling for grade exports or reconciliation instead of manual arithmetic.
- Use the Computer Use plugin for live StudyDesk or institutional web workflows.
- If another staff member's approval is required, stop at the ready-to-approve state and report exactly what remains.
