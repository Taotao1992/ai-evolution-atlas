# Teaching Video Editing

## Overview

Use this skill for the Panopto tutor-video workflow shown in the UniSQ guide. The recorded workflow is: open `My Folder`, upload the recording, wait for processing, edit the Panopto item, import automatic captions, add contents markers, apply cuts only where truly needed, then share the link into StudyDesk with the right visibility.

## Workflow

1. Confirm the target and intended outcome.
   Identify the exact Panopto item, course/week, and whether the user wants a dry run, a Panopto-only change, or the full Panopto-plus-StudyDesk publishing flow.

2. Review the reference guide.
   Read [references/panopto-guide.md](references/panopto-guide.md).
   The reference includes extracted workflow notes from the actual recording, not just the title.

3. Open Panopto and place the recording correctly.
   Sign in with the UniSQ account, go to `My Folder`, and create or confirm the course-specific folder before uploading.
   If the recording is still processing and edit controls are incomplete, wait until the viewer/edit state is ready.

4. Apply the core Panopto edits in the recorded order.
   Open `Edit`.
   In `Settings`, update the title if needed and untick the discussion/comment options so students cannot leave comments under the video.
   Treat the copyright notice as optional unless the user specifically wants it.
   In `Captions`, import automatic captions. The guide treats minor caption errors as acceptable unless the user explicitly asks for cleanup.
   In `Contents`, add chapter markers by placing the marker at the right time and typing the section title.

5. Cut only the parts that should not be shown.
   Use cuts for genuinely wrong, redundant, or misleading segments.
   Do not cut ordinary exercise pauses or short student-thinking time unless the user asks.
   The recorded guide prefers cutting unwanted sections before doing final timing cleanup on contents.
   Apply the edits and wait for Panopto to process again.

6. Share the video the way the recording demonstrates.
   Open `Share`, change the visibility to the UniSQ organization scope used in the guide so students with UniSQ accounts can open the link, then copy the link.
   In StudyDesk, turn editing on, add a `URL` activity in the correct section, paste the Panopto link, name it clearly, return to the course, move it into place, and make sure it is not hidden.

7. Verify from the student side.
   Re-open the Panopto item and the StudyDesk link.
   Check title, folder, playback, captions/contents presence, sharing scope, and that the StudyDesk activity is visible in the intended week.

## Safety Checks

- Stop before delete, replace, or permanent transcript/caption removal unless the user clearly confirms.
- Stop before widening access beyond the UniSQ organization or changing an already-live StudyDesk link that students rely on unless the user explicitly wants that.
- Treat comment settings, sharing scope, and student visibility as separate checks; do not assume one change preserved the others.

## Execution Notes

- Use the Computer Use plugin for live browser work.
- If the user only wants a checklist or a dry run, stop before live Panopto or StudyDesk changes.
- If the live UI wording differs from the recording, follow the current UI labels and mention the mismatch.
