# Teaching Late Penalty Turnitin Results

## Overview

Handle penalties, Turnitin review, and result release as one controlled workflow. Confirm the course policy first, separate automatic checks from judgment calls, and stop before release if the cohort still has unresolved exceptions.

## Workflow

1. Confirm policy inputs.
   Collect the due date and time, approved extensions or exemptions, late-penalty formula, cohort scope, and intended release window.

2. Review the reference guide.
   Read [references/panopto-guide.md](references/panopto-guide.md).
   If the Panopto guide is inaccessible, use the checklist below and ask for policy details that cannot be safely inferred.

3. Gather submission evidence.
   Check submission timestamps, extension records, exempt students, missing submissions, and Turnitin report status before changing grades or releasing results.

4. Apply or prepare late penalties.
   Compute penalties from the stated policy, keep before/after values, and separate manual exceptions from standard cases.

5. Review Turnitin carefully.
   Identify missing reports, unusual similarity, or obvious false positives, but do not treat a percentage alone as an academic-integrity conclusion.

6. Pause before releasing results.
   Reconfirm the exact activity, cohort, and release intent immediately before the live release step.

7. Verify the release outcome.
   Re-open the activity or relevant results view and confirm the expected students can see the intended final result state.

## Safety Checks

- Do not invent or approximate the penalty formula.
- Do not release results while unresolved penalty exceptions or Turnitin concerns remain.
- Distinguish administrative checking from formal misconduct escalation.

## Execution Notes

- Use spreadsheet tooling when penalties need batch calculation or audit output.
- Use the Computer Use plugin for live browser actions in StudyDesk or related systems.
- If the user only wants preparation, stop at a reviewed checklist or ready-to-release summary.
