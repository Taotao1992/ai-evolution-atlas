# Teaching Video Editing

## Overview

Use the institutional guide when available, then make the smallest safe edit that satisfies the request. Prefer reversible changes, verify playback and visibility after each edit, and keep a short audit trail of what changed.

## Workflow

1. Confirm the task scope.
   Identify the exact video, course or folder, requested edits, deadline, and whether the task is metadata-only or timeline/content editing.

2. Review the reference guide.
   Read [references/panopto-guide.md](references/panopto-guide.md).
   If access is blocked, continue with the checklist below and surface any missing institutional details instead of guessing.

3. Capture the starting state.
   Record the current title, folder, availability, captions status, and any requested trim points or notes.
   If more than one similar video exists, verify the exact target before editing.

4. Apply edits in small steps.
   Use browser automation when the user wants the live Panopto item updated.
   Prefer one category of change at a time: trim, retitle, move, availability, captions.
   Avoid deleting or replacing the source video unless the user explicitly asks.

5. Verify the result.
   Re-open the edited video and check playback start/end, audio, title, folder, captions, and student visibility or sharing settings.

6. Report the outcome.
   Summarize what changed, what was verified, and any remaining manual follow-up.

## Safety Checks

- Stop before destructive actions such as delete, replace, or permanent transcript removal unless the user clearly confirms.
- Treat captions and availability as separate verification steps; do not assume they remain correct after editing.
- If the guide and live UI disagree, trust the live UI labels but mention the mismatch.

## Execution Notes

- Use the Computer Use plugin for live browser work.
- If the user only wants a checklist or a dry run, prepare the plan without touching the live system.
- If the user provides a second Panopto or written guide later, merge it into the workflow before continuing.
